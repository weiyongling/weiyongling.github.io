第一章 计算机网络体系结构

1. 计算机网络的组成与工作方式

  • 边缘部分：由所有连接在互联网上的主机（端系统）组成。用户直接使用，用来进行通信和资源共享。其通信方式分为 C/S 方式（客户/服务器）和 P2P 方式（对等连接）。
  • 核心部分：由大量网络和连接这些网络的路由器组成。为边缘部分提供连通性和交换服务。核心部分最关键的交互方式是分组交换（Packet Switching）。

2. 三种交换方式的对比（大题常考常识）

  • 电路交换：整个报文从源点到终点连续地传输。需要经历：建立连接（占用资源） → 通话 → 释放连接。不适合突发突发流量。
  • 报文交换：整个报文先传送到相邻结点，全部存储下来后查找转发表，再转发到下一个结点（存储-转发）。
  • 分组交换：把长报文划分为若干个等长的、较短的“分组”。单个分组逐个地在网络中存储-转发。优点：高效（动态分配带宽）、灵活、迅速、可靠。缺点：会产生排队时延，且可能出现丢包和乱序。

3. 计算机网络的性能指标

四种时延（总时延 = 发送时延 + 传播时延 + 处理时延 + 排队时延）：

1. 发送时延（传输时延） = 数据帧长度(bit) / 发送速率(bit/s) [发生在网络适配器内部]
2. 传播时延 = 信道长度(m) / 电磁波在信道上的传播速率(m/s) [发生在传输媒体上]
  • 时延带宽积：时延带宽积 = 传播时延 × 带宽。表示一个链路容纳比特的最大能力，又称“以比特为单位的链路长度”。
  • 往返时间 RTT：从发送方发送数据开始，到发送方收到接收方的确认所经历的时间。

4. 体系结构的分层模型

体系结构主要有三种：OSI（七层）、TCP/IP（四层）、教学用五层模型。

五层模型及其数据对等单位：
1. 应用层：通过应用进程间的交互完成特定网络应用（数据单位：报文 Message）
2. 运输层：向两台主机中进程之间的通信提供通用的数据传输服务（数据单位：TCP段 Segment / UDP数据报 Datagram）
3. 网络层：负责为分组交换网上的不同主机提供通信服务，选择路由（数据单位：IP数据报/分组 Packet）
4. 数据链路层：将网络层交下来的IP数据报组装成帧（数据单位：帧 Frame）
5. 物理层：透明地传送比特流（数据单位：比特 Bit）

第二章 物理层原理及计算

1. 物理层四大特性

  • 机械特性：指明接口所用接线器的形状、尺寸、引脚数及排列方式。
  • 电气特性：指明在接口电缆的各条线上出现的电压的范围。
  • 功能特性：指明某条线上出现的某一电平的电压的意义。
  • 过程特性：指明对于不同功能的各种可能事件的出现顺序。

2. 限制码元传输速率的两个核心定理（必考计算题）

(1) 奈氏准则（Nyquist）：在理想低通信道中，最高码元传输速率 = 2W（Baud），其中W是信道带宽（Hz）。

理想低通信道极限数据率 = 2W * log2(V) (单位: bit/s)
* V 表示码元的状态数（即调制电平数）

(2) 香农公式（Shannon）：在带噪信道中，极限信息传输速率 C 满足：

C = W * log2(1 + S/N) (单位: bit/s)
* W 为信道带宽(Hz)；S 为信道内所传信号的平均功率；N 为信道内部的高斯噪声功率。
* 注意：题目中给出的信噪比通常是分贝(dB)，公式为: dB = 10 * log10(S/N)。例如 30dB 意味着 S/N = 1000。

3. 编码与调制

  • 曼彻斯特编码：每个码元正中间都有一次电压跳变。位周期中心的上跳变代表0（或1），下跳变代表1（或0）。自带同步信号，但占用的频带宽度加倍。
  • 差分曼彻斯特编码：每个码元中心仍有跳变，但码元起始处有跳变代表0，无跳变代表1。抗干扰性比曼彻斯特编码更好。

第三章 数据链路层与局域网

1. 三个基本问题

  • 封装成帧：在一段数据的前后分别添加首部（SOH）和尾部（EOT），构成一个帧。确定帧的界限。
  • 透明传输：解决数据中偶然出现与帧定界符相同的控制字符的问题。使用字节填充（Byte Stuffing）或比特填充（0比特插入法，5个1加一个0）。
  • 差错检测：广泛采用循环冗余检验（CRC）。利用多项式除法算出余数作为冗余码（FCS）拼接在帧尾。接收方收到后除以同样的多项式，若余数为0则认为无差错。注意：CRC只能做到无差错接受，无法保证可靠传输。

2. 局域网核心协议：CSMA/CD

主要用于总线形以太网。核心口诀：先听后发，边听边发，冲突停发，随机重发。

  • 争用期（碰撞窗口）：单程传播时延为 τ，争用期大小为 2τ。发送方在发送数据后经过 2τ 的时间仍未检测到碰撞，才能确认这次发送绝对不会发生碰撞。
  • 最短有效帧长：以太网规定的最短有效帧长为 64 字节。凡长度小于 64 字节的帧都是由于冲突而异常中止的无效帧。
  • 截断二进制指数退避算法：发生碰撞后，停止发送，推迟一个随机时间再发送。
退避时间 = 争用期 2τ * r
其中 r = 随机数从 [0, 1, ..., (2^k - 1)] 中选择，k = min(重传次数, 10)。当重传达 16 次仍不能成功时溢出报错。

3. 以太网交换机与 VLAN

  • 交换机原理：工作在数据链路层。拥有自学习能力，通过检查收到的帧的源 MAC 地址来自动建立和维护“转发表”（MAC地址-接口映射）。
  • 虚拟局域网 (VLAN)：通过在以太网帧结构中插入 4 字节的 802.1Q 标签（其中有 12 位 VLAN ID），在逻辑上将局域网划分为多个广播域。
  • 接口类型：Access 接口（只能属于一个VLAN，常连接电脑）；Trunk 接口（允许多个VLAN通过，常用于交换机之间的互连）。

第四章 网络层与 IP 编址

1. 传统分类 IP 地址

IP 地址由网络号（net-id）和主机号（host-id）组成。总长 32 位。

  • A类地址：0.0.0.0 ~ 127.255.255.255（网络号占8位，第一位固定为0）。127.X.X.X 为本地环回地址。
  • B类地址：128.0.0.0 ~ 191.255.255.255（网络号占16位，前两位固定为10）。
  • C类地址：192.0.0.0 ~ 223.255.255.255（网络号占24位，前三位固定为110）。
  • D类地址：224.0.0.0 ~ 239.255.255.255（多播/组播地址）。

注意不能分配的主机号：主机号全0代表本网络网络地址，主机号全1代表本网络的广播地址。

2. CIDR（无分类编址）与子网划分计算

斜线记法（如 192.168.1.0/24），后面的 24 表示网络前缀占 24 位，对应的子网掩码就是 24 个 1 和 8 个 0，即 255.255.255.0。

🔥 必考大题：子网划分与最长前缀匹配
1. 路由聚合（超网）：把网络前缀相同的连续网络合并。找相同的二进制前缀，相同部分作为新的前缀长度。
2. 最长前缀匹配：路由器在转发表中查找路由时，可能会匹配到多个项目。应该选择网络前缀最长（掩码1最多）的那一条，因为这样路由最精确。

3. 网络层四大核心协议

  • IP 协议：核心协议，提供无连接、尽最大努力交付的服务。
  • ARP 协议（地址解析协议）：将 IP 地址转换为 MAC 地址。工作原理：主机通过在局域网内广播发送 ARP 请求分组，目标主机检测到后通过单播回复 ARP 响应分组。从而填充自己的 ARP 缓存表。
  • ICMP 协议（网际控制报文协议）：允许路由器或主机报告差错情况和提供异常情况的报告。典型应用包括：PING（使用 ICMP 回送请求与回答报文）和 Traceroute（用来跟踪分组经过的路由）。

4. 路由协议（内部网关协议 IGP）

  • RIP 协议：基于距离向量算法。使用 UDP（端口520）传输。最大跳数为 15 跳，16 跳表示不可达。坏消息传得慢（存在路由环路问题）。
  • OSPF 协议：基于链路状态算法。直接使用 IP 数据报传输。使用 Dijkstra 算法计算最短路径树。全网独立自适应，收敛速度快，无路由环路。

第五章 运输层（双核心协议）

1. UDP 与 TCP 的对比

  • UDP：无连接、尽最大努力交付、面向报文（不拆分不合并，保留应用层报文边界）、支持一对一/一对多/多对多交互、首部开销小（仅 8 字节）。
  • TCP：面向连接、可靠交付、全双工通信、面向字节流（把应用层交下来的数据看成无结构的字节流，可拆分或重组）、首部开销大（至少 20 字节）。

2. TCP 的三次握手（必须死记硬背状态名和大题图解）

  1. 第一次：客户机向服务器发送连接请求。首部：SYN=1, seq=x。客户端进入 SYN-SENT 状态。
  2. 第二次：服务器收到后同意连接，回复确认。首部：SYN=1, ACK=1, seq=y, ack=x+1。服务器进入 SYN-RCVD 状态。
  3. 第三次：客户机给出最终确认。首部：ACK=1, seq=x+1, ack=y+1。双方进入 ESTABLISHED 状态，连接建立。

3. TCP 的四次挥手

  1. 第一次：客户机发起释放连接。首部：FIN=1, seq=u。进入 FIN-WAIT-1 状态。
  2. 第二次：服务器回复确认。首部：ACK=1, seq=v, ack=u+1。服务器进入 CLOSE-WAIT（半关闭），客户端收到后进入 FIN-WAIT-2。
  3. 第三次：服务器数据发完，发起释放连接。首部：FIN=1, ACK=1, seq=w, ack=u+1。进入 LAST-ACK 状态。
  4. 第四次：客户机回复最终确认。首部：ACK=1, seq=u+1, ack=w+1。客户机进入 TIME-WAIT 状态。必须等待 2MSL（最长报文段寿命）时间后才进入 CLOSED。服务器收到该 ACK 后直接 CLOSED。

为什么要有 TIME-WAIT 2MSL？1. 确保第四次挥手的 ACK 报文能最终到达服务器。2. 防止本连接中产生的“已失效的连接请求报文段”出现在下一个新连接中。

4. TCP 拥塞控制四大算法（核心大题：绘图分析）

发送方维持一个拥塞窗口 cwnd。发送窗口 = min(拥塞窗口, 接收方通告的接收窗口 rwnd)。

  • 慢开始：由小到大指数级增长 cwnd（1, 2, 4, 8...），直到达到慢开始门限 ssthresh。
  • 拥塞避免：到达 ssthresh 后，改为线性加性增大（每次往返时间 RTT 只让 cwnd + 1）。
  • 网络拥塞处理：只要出现超时（丢包），立即将新门限 ssthresh 设为当前 cwnd 的一半，然后将 cwnd 降为 1，重新开始慢开始。
  • 快重传与快恢复：接收方一旦收到失序段，立即发送对已收到的重复确认。发送方只要连续收到3个重复确认，判定报文丢失，执行快重传（立即重传丢失段），不执行慢开始，而是执行快恢复：把 ssthresh 减半，并将 cwnd 设为减半后的值，直接开始拥塞避免（线性增长）。

第六章 应用层核心协议

1. DNS 域名解析系统

  • 概念：工作在 UDP 上，端口 53。负责将人类易读的域名转换为机器可识别的 IP 地址。
  • 查询方式：递归查询（靠别人查，主机问本地域名服务器常用）；迭代查询（本地域名服务器亲自出马，逐级询问根、顶级、权限域名服务器）。

2. HTTP 超文本传输协议

  • 核心机制：工作在 TCP 上，端口 80。
  • HTTP 1.0 与 1.1：1.0 默认采用非持续连接（每次下载对象都需要建立 TCP 三次握手）；1.1 默认采用持续连接（Keep-Alive，可以在一个连接上连续发送多个对象），并支持管道化传输。
  • 常用状态码：
    • 200 OK：请求成功。
    • 301 Moved Permanently：永久重定向；302 Found：临时重定向。
    • 400 Bad Request：客户端请求语法错误；403 Forbidden：服务器理解但拒绝服务。
    • 404 Not Found：找不到服务器资源。
    • 500 Internal Server Error：服务器内部错误。

3. DHCP 动态主机配置协议

使用 UDP 传输，端口 67(服务器) / 68(客户端)。即插即用自动分配 IP 地址。

四步骤（全是广播形式交互）：

  1. DHCP Discover（客户端全网广播找 DHCP 服务器）
  2. DHCP Offer（服务器全网广播提供可用 IP 方案）
  3. DHCP Request（客户端全网广播选择其中一个 IP 方案）
  4. DHCP ACK（服务器全网广播确认该 IP 租约生效）
